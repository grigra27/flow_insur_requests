Тема: Обсуждение нового формата передачи страховых заявок

Добрый день.

Хотели бы предложить обсудить возможность изменения текущего формата передачи страховых заявок.

Сейчас заявки передаются в виде Excel-файлов. Этот формат удобен для ручного заполнения и визуальной проверки, но при автоматической загрузке у него есть ряд ограничений:

- данные завязаны на конкретные ячейки и положение строк;
- для заявок ИП структура сдвигается, из-за чего требуется отдельная логика обработки;
- в многообъектных заявках несколько предметов страхования фактически представлены как таблица, но при загрузке их сложнее корректно сохранить как отдельные объекты;
- часть признаков определяется не как явные значения, а по наличию отметок или текста в отдельных ячейках;
- при изменении шаблона Excel даже небольшое смещение строк или объединение ячеек может приводить к ошибкам распознавания;
- часть полезных данных сложнее автоматически валидировать до загрузки.

Мы предлагаем рассмотреть переход на структурированный формат выгрузки заявки из вашей системы. Предпочтительный вариант для нас - JSON по согласованной схеме. Такой формат позволит передавать заявку не как визуальную анкету, а как набор точных структурированных данных: клиент, параметры сделки, условия страхования, список объектов страхования, варианты франшизы, дополнительные риски и комментарии.

Для вас такой подход тоже может быть полезен:

- меньше ручных уточнений по заявкам после отправки;
- меньше риск, что часть данных будет не распознана или будет распознана неоднозначно;
- быстрее первичная обработка и передача заявки в работу;
- можно заранее проверять корректность выгрузки по схеме до отправки;
- многообъектные заявки будут передаваться без потери структуры;
- формат будет менее зависим от визуального шаблона Excel и его будущих изменений;
- при необходимости такой же формат можно использовать не только для файловой передачи, но и для дальнейшей интеграции через API.

Во вложении направляем материалы для обсуждения:

1. JSON-схему предлагаемого формата заявки.
2. Три примера исходных Excel-заявок.
3. Три соответствующих примера заполненных JSON-файлов на основе этих заявок:
   - обычная заявка по КАСКО/спецтехнике;
   - многообъектная заявка;
   - заявка по страхованию имущества с дополнительным риском перевозки.

Идеальным для нас вариантом был бы JSON в приложенной структуре. При этом мы готовы рассмотреть и другие варианты, если для вашей системы они будут проще в реализации: например XML, строгий XLSX-шаблон без объединенных ячеек или другой структурированный формат. Главное, чтобы в новом формате данные передавались явно, без привязки к координатам ячеек и без необходимости распознавать смысл по визуальному расположению отметок.

Будем признательны, если вы сможете оценить, насколько реалистична такая выгрузка с вашей стороны, и какой из вариантов формата был бы для вас наиболее удобен.
