Справка по работе со сводами

Как устроены своды сейчас

Ключевое: свод создается из карточки заявки кнопкой «Создать свод», а не автоматически.

Где создается свод

  • Кнопка «Создать свод» доступна в карточке заявки, если у заявки статус: «Загружено», «Письмо сгенерировано» или «Письма отправлены».
  • Если свод уже существует, открывается существующий свод.
  • При создании из статуса заявки «Загружено» заявка переводится в «Письмо сгенерировано».

Кто имеет доступ

  • Страницы сводов доступны пользователям групп «Администраторы» и «Пользователи».
  • Страница статистики сводов доступна только группе «Администраторы».

Список сводов

Что есть на странице

  • Вкладки филиалов: «Все» + список филиалов из текущих сводов.
  • Фильтры: номер ДФА, месяц создания, год создания.
  • Пагинация по 30 сводов на страницу.
  • В строке свода отображаются: заявка, клиент, филиал, статус, дата, число компаний.
Важно: счетчик «Компаний» в своде считает уникальные страховые компании, а не количество строк предложений.

Ручное добавление предложений

Основные правила

  • В интерфейсе кнопка ручного добавления показывается в статусе «Сбор предложений».
  • Компания выбирается из закрытого списка (включая «Другое»).
  • Для одного свода нельзя создать дубликат: одна компания + один год страхования.

Валидация полей

  • Год страхования: от 1 до 10.
  • Страховая сумма: больше 0.
  • Франшизы: не могут быть отрицательными.
  • Нужно указать хотя бы одну премию (вариант 1 или вариант 2).
  • Премия не может быть больше страховой суммы.
  • Рассрочка по ручной форме: 1, 2, 3, 4 или 12 платежей в год.

Загрузка ответов из Excel

Загрузка работает только в статусе «Сбор предложений».

Ограничения загрузки

  • Только .xlsx.
  • До 10 файлов за загрузку.
  • До 1 MB на файл.
  • До 10 MB суммарно.

Какие ячейки читает система

Данные Ячейка Комментарий
Название компании B2 Сопоставляется с закрытым списком компаний. Если не найдено совпадение, ставится «другое».
Примечание для 1-го года F2 Записывается в комментарий предложения первого года.
Год страхования A6:A10 Допустимые значения: 1..10.
Страховая сумма B6:B10 Обязательное поле строки.
Премия-1 D6:D10 Обязательное поле строки.
Франшиза-1 E6:E10 Обязательное поле строки (текст в ячейке интерпретируется как 0).
Рассрочка-1 F6:F10 Допустимо: 1, 2, 3, 4, 6, 12.
Премия-2 / Франшиза-2 / Рассрочка-2 H:I:J Необязательные поля второго варианта.
Практический нюанс: первичная проверка формы ожидает данные минимум в строке 6 или 7. Если файл заполнен только с 8-й строки, загрузка будет отклонена.

Что происходит при загрузке

  • Для каждого года создается отдельное предложение.
  • Если в своде уже есть такая же пара «компания + год», файл/строка отклоняется как дубликат.
  • По множественной загрузке выдается результат по каждому файлу: успешные и ошибки.

Статусы и завершение

Статусы свода

Статус Что обычно делают Ограничения
Сбор предложений Загрузка Excel и ручное наполнение Только в этом статусе доступна загрузка ответов компаний
Готов к отправке Финальная проверка перед выгрузкой Только в этом статусе доступна генерация Excel
Отправлен в Альянс Фиксация факта отправки Записывается дата/время отправки
Завершен: акцепт/распоряжение Фиксация выбранной СК Обязателен выбор компании из предложений свода
Завершен: не будет Закрытие без акцепта Поля выбранной компании очищаются

Выбор СК и варианта при акцепте

  • Компания выбирается только из тех, что уже есть в предложениях текущего свода.
  • Если у выбранной компании доступны оба варианта (1 и 2) по всем годам, выбор варианта обязателен.
  • Если доступен только один вариант, система проставляет его автоматически.

Резюме по сделке

  • Ссылка «Резюме по сделке» доступна, когда свод завершен с акцептом и выбрана СК.
  • В резюме показываются данные заявки и предложения выбранной компании по годам.
Автозакрытие: в проекте есть команда автоматического закрытия старых сводов (`auto_close_stale_summaries`). По умолчанию она переводит своды старше 30 дней из статусов collecting/ready/sent в «Завершен: не будет» (если настроен cron/запуск команды).

Выгрузка Excel

Доступно только в статусе «Готов к отправке».

Варианты выгрузки

  • Полный Excel — с техническим листом (`tech_info`).
  • Клиентский Excel — без технического листа.

Как выбирается шаблон

  • Если в своде есть хотя бы одно предложение с заполненной премией-2 (> 0), используется полный шаблон с колонками двух вариантов.
  • Если премии-2 нет ни у одного предложения, используется упрощенный шаблон.

Что проверяется перед генерацией

  • Наличие связанной заявки.
  • Заполненность обязательных полей заявки: номер ДФА, клиент, предмет лизинга.

Формат имени файла

Полный: full_svod_<цифры_из_DFA>_<дд_мм_гггг>.xlsx
Клиентский: client_svod_<цифры_из_DFA>_<дд_мм_гггг>.xlsx

Частые ошибки и что делать

1) «Загрузка ответов компаний доступна только при статусе "Сбор предложений"»

Переведите свод в статус «Сбор предложений» в блоке «Управление статусом».

2) «Предложение от компании ... на ... год уже существует»

В одном своде нельзя иметь две записи одной компании на один и тот же год. Отредактируйте существующую запись или измените год.

3) Ошибка структуры файла Excel

Проверьте: формат .xlsx, компания в B2, данные годов в строках 6-10, обязательные колонки A/B/D/E/F.

4) «Файл можно генерировать только для сводов в статусе "Готов к отправке"»

Для выгрузки сначала смените статус на «Готов к отправке».

5) Нет кнопки «Резюме по сделке»

Нужно перевести свод в статус «Завершен: акцепт/распоряжение» и выбрать страховую компанию (и вариант, если требуется).
